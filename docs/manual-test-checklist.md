# Workbuddian 手动测试清单

> 版本：v0.3.x（main 最新）· 日期：2026-07-12
> 用法：在 Obsidian 里 **Cmd+R 重载**后，逐项操作并打勾。每项给了「操作」和「预期」。
> 发现问题就在该项下写下现象（含控制台 `[BB]` 日志），发给我修。
>
> 打开开发者控制台：**Cmd + Opt + I**（Console 面板，过滤 `[BB]`）。

---

## 0. 部署与加载

- [ ] **插件已加载最新代码**
  - 操作：设置 → 第三方插件 → 确认 Workbuddian 已启用；`Cmd+R` 重载一次。
  - 预期：无报错弹窗；控制台无红色异常堆栈。

- [ ] **控制台初始无错误**
  - 操作：打开控制台看 Console。
  - 预期：只有 `[BB]` 之类普通 log，没有 `Workbuddian 加载失败` 或红色 Error。

---

## 1. 打开面板

- [ ] **Ribbon 图标打开**
  - 操作：点左侧栏的小猪图标。
  - 预期：右侧栏打开聊天面板，占满侧栏高度。

- [ ] **命令面板打开**
  - 操作：`Cmd+P` → 运行「Workbuddian: 打开聊天面板」。
  - 预期：同上，聚焦到面板。

- [ ] **主编辑区大面板**
  - 操作：`Cmd+P` → 「Workbuddian: 在主编辑区打开大面板」。
  - 预期：在主编辑区以全宽标签页打开另一个聊天面板。

- [ ] **双面板不互相冲掉**
  - 操作：侧栏 + 主编辑区各开一个，在其中一个发消息 / 新建对话。
  - 预期：两个面板各自状态正常，不会用旧快照覆盖对方。

---

## 2. 基本聊天与流式

- [ ] **发送与流式输出**
  - 操作：输入「你好，简单介绍下自己」，按 Enter。
  - 预期：先出现「思考中…」占位，随后**逐字流式**显示回复；结束后占位消失。

- [ ] **思考块**
  - 操作：发一个需要推理的问题。
  - 预期：出现可折叠「思考中/已思考」块，点击标题能展开/收起。

- [ ] **工具调用块**
  - 操作：让它读一个 vault 文件（如「读一下 xxx.md 讲了什么」）。
  - 预期：出现「工具调用」块，列出工具名+参数，图标随读/写/搜索变化。

- [ ] **Markdown 渲染**
  - 操作：让它输出代码块、表格、列表、引用。
  - 预期：均正确渲染（代码高亮、表格边框、列表缩进、引用左边框）。

- [ ] **停止生成**
  - 操作：发一个长回复，回复中途点发送键（此时是 ■ 停止图标）。
  - 预期：生成立即中断，按钮变回纸飞机。

- [ ] **Enter / Shift+Enter**
  - 操作：输入多行，用 Shift+Enter 换行；单独 Enter 发送。
  - 预期：Shift+Enter 换行不发送；Enter 发送。

---

## 3. 输入框与工具栏布局

- [ ] **整框 + 聚焦高亮**
  - 操作：点进输入框。
  - 预期：输入框是**一个带边框的整框**，聚焦时**整框**边框高亮（不是只有 textarea）。

- [ ] **底部工具栏一行**
  - 预期：框内底部有一行：`[模型名] [📎] [🛡] … [◔? 无] [纸飞机]`（圆环用量已移除，不应出现）。

- [ ] **高度合适**
  - 预期：输入区不臃肿；多行输入时 textarea 自动增高（有上限）。

- [ ] **发送键永不被挤出**
  - 操作：把侧栏拖到**很窄**。
  - 预期：模型名会被压缩/截断，但**纸飞机发送键始终留在框内**，不会跑出去。

---

## 4. 模型下拉

- [ ] **点击弹出（非悬停）**
  - 操作：把鼠标停在模型名上**不点**。
  - 预期：**不**自动弹菜单（悬停弹出已取消，避免误触）。
  - 操作：**点击**模型名。
  - 预期：弹出模型菜单（auto + 9 个模型），当前项有勾。

- [ ] **切换生效 + 持久化**
  - 操作：选一个别的模型（如 `glm-5.2`），发一条消息；然后关面板重开。
  - 预期：按钮显示新模型；重开后仍是新模型（已存设置）。

---

## 5. 附件 📎（系统文件选择器）

- [ ] **打开系统选择器**
  - 操作：点 📎。
  - 预期：弹出**操作系统的文件选择器**（可选任意文件，不限 vault）。

- [ ] **选中显示 chip**
  - 操作：选 1~2 个文件。
  - 预期：输入框**上方**出现附件 chip（显示文件名），有 ✕。

- [ ] **删除 chip**
  - 操作：点 chip 的 ✕。
  - 预期：该附件移除。

- [ ] **发送注入 + 用后清空**
  - 操作：带着附件发一条「这个文件讲了什么」。
  - 预期：AI 能读到文件内容（或按路径读取）；发送后附件 chip 清空。
  - 备注：若 AI 说读不到 vault 外的文件，把现象记下（可能要改成读文件内容注入）。

---

## 6. 授权 🛡（permission 模式）

- [ ] **菜单只有两项 + 各带图标**
  - 操作：点 🛡。
  - 预期：菜单只有「默认」「完全访问」两项，**各自图标不同**（默认=盾牌，完全访问=盾牌带感叹号）。

- [ ] **切到「完全访问」→ 按钮图标变**
  - 操作：选「完全访问」。
  - 预期：工具栏的盾牌图标变成**内含「!」的盾牌**（shield-alert）；tooltip 显示当前模式。

- [ ] **切回「默认」→ 图标复原**
  - 预期：盾牌恢复普通样式。

- [ ] **持久化**
  - 操作：设为某模式后重开面板。
  - 预期：图标/模式保持。

---

## 7. 发送图标

- [ ] **图标样式**
  - 预期：发送是**纸飞机图标、无方块底色**（强调色着色）。

- [ ] **流式时变停止**
  - 操作：发消息，观察生成期间。
  - 预期：变成 ■ 停止图标；结束后变回纸飞机。

---

## 8. 选区注入（笔记选中 → 聊天）

- [ ] **实时出现（无需点输入框）**
  - 操作：打开一篇笔记，**选中一段文字**（聊天面板开着）。
  - 预期：聊天输入框**上方立刻**出现「选区」chip（显示笔记名 + 预览）。

- [ ] **取消选择即消失**
  - 操作：在笔记里点一下取消选中。
  - 预期：选区 chip 消失。

- [ ] **发送注入**
  - 操作：选中一段，问「这段讲了什么」并发送。
  - 预期：AI 能看到那段选中文字（只读参考）。

- [ ] **切换选区实时更新**
  - 操作：改选另一段。
  - 预期：chip 内容跟着变。

- [ ] **不干扰 Inline Edit**
  - 预期：选区注入只影响聊天，不影响「用 CodeBuddy 编辑选区」命令。

---

## 9. @ 笔记引用（原有功能回归）

- [ ] **@ 触发选择器**
  - 操作：输入框里打 `@` + 笔记名片段。
  - 预期：弹出笔记候选下拉。

- [ ] **插入 chip**
  - 操作：选一个笔记。
  - 预期：输入框上方出现 `@[[笔记]]` 引用 chip，可 ✕ 删除。

- [ ] **发送注入笔记正文**
  - 预期：AI 能读到被引用笔记的内容。

---

## 10. 斜杠命令

- [ ] **`/` 自动补全**
  - 操作：输入 `/`。
  - 预期：弹出命令候选（内置命令 + `.codebuddy/commands` 自定义命令）。

- [ ] **`/clear` 本地新建**
  - 操作：发送 `/clear`。
  - 预期：本地新建一个空对话（不发 CLI）。

- [ ] **其它 `/` 命令透传**
  - 操作：发送 `/status` 或 `/cost`。
  - 预期：原样透传给 CLI，返回 CLI 的结果（不注入 vault 上下文）。

---

## 11. 配色

- [ ] **默认土黄 + 黑字**
  - 前提：设置里主色调为空（没手动调过；若调过，先到 设置→外观→主色调 右边点 ↺ 恢复默认）。
  - 预期：**用户气泡=土黄底 + 黑字**；激活的对话标签=土黄底 + 黑字；发送纸飞机=土黄色；输入框聚焦环=土黄色。

- [ ] **自定义主色调**
  - 操作：设置→外观→聊天主色调 选一个颜色。
  - 预期：气泡/按钮/边框实时变该色。

- [ ] **恢复默认**
  - 操作：点主色调右边 ↺。
  - 预期：回到土黄默认。

---

## 12. 界面语言

- [ ] **切 English**
  - 操作：设置→外观→界面语言 选 English。
  - 预期：设置页文案**立即**变英文；弹出提示「重开面板生效」。

- [ ] **面板变英文**
  - 操作：重开聊天面板（或 Cmd+R）。
  - 预期：面板 UI（发送、占位符、标签、菜单等）为英文。

- [ ] **切回中文 / Auto**
  - 预期：同理恢复；Auto 跟随 Obsidian 界面语言。

---

## 13. 设置页

- [ ] **分组正确**
  - 预期：分组含「CodeBuddy 连接 / 上下文注入 / 外观 / 重置 / 备份」。

- [ ] **已无「模型 / 授权」**
  - 预期：设置页里**看不到**模型下拉和授权下拉（都移到工具栏了）。

- [ ] **外观组有 语言 + 主色调**
  - 预期：能看到「界面语言」「聊天主色调」。

- [ ] **连接项可用**
  - 操作：改 CodeBuddy 路径 / Node 路径 / 超时。
  - 预期：可保存；超时只接受正整数。

- [ ] **上下文注入开关**
  - 操作：切「注入 Vault 上下文」「注入当前笔记链接」。
  - 预期：开关生效并持久化。

- [ ] **重置为默认**
  - 操作：点「重置为默认」→ 再点一次确认。
  - 预期：所有设置回默认，提示「已重置」。

- [ ] **导出 / 导入设置**
  - 操作：点「复制为 JSON」；再把 JSON 粘贴进导入框点确认。
  - 预期：导出到剪贴板；导入成功刷新；乱码 JSON 导入报「解析错误」不崩。

## 14. 对话管理

- [ ] **新建对话** — 点 `+`，出现新空对话。
- [ ] **切换标签** — 点不同标签，消息区切换对应对话。
- [ ] **重命名** — 双击标签标题，改名并保存。
- [ ] **删除对话** — 关闭某对话，激活切到另一个。
- [ ] **搜索** — 点搜索图标，输入关键词，按标题/正文过滤。
- [ ] **导出为笔记 / 复制** — 标签右键菜单，导出为笔记或复制到剪贴板。

---

## 15. 会话持久化

- [ ] **重启后恢复**
  - 操作：发几条消息 → 完全退出并重开 Obsidian → 打开面板。
  - 预期：对话历史与内容都还在。

- [ ] **上下文用量数据仍在采集（隐性）**
  - 说明：圆环 UI 已移除，但底层仍存 `lastUsage`；此项无需可见验证，跳过即可。

## 16. Inline Edit（编辑选区）

- [ ] **命令触发**
  - 操作：在笔记里选中一段 → `Cmd+P` →「用 CodeBuddy 编辑选区」。
  - 预期：弹出指令 Modal。

- [ ] **diff 预览 + 接受/拒绝**
  - 操作：输入「改简洁」→ 确认。
  - 预期：弹出行级 diff（增/删高亮）；「接受」写回选区，「拒绝」不改。

---

## 17. 错误处理

- [ ] **错误卡片**
  - 操作：把 CodeBuddy 路径故意改错 → 发消息。
  - 预期：显示 ⚠️ 错误卡片（文案 + [重试] [打开设置]），不是纯文本。

- [ ] **重试**
  - 操作：把路径改回，点错误卡片「重试」。
  - 预期：用同一条消息重发成功。

- [ ] **打开设置**
  - 操作：点错误卡片「打开设置」。
  - 预期：跳到 Workbuddian 设置页。

---

## 18. 「无响应」诊断（本轮修复）

- [ ] **正常回复不误报**
  - 操作：正常聊天。
  - 预期：不会无故显示「(无响应，请重试)」。

- [ ] **若出现，控制台有诊断**
  - 操作：万一出现「(无响应)」，看控制台。
  - 预期：有 `[BB] empty response — chunks:{...} | resultLen:N`；把该行发我判断类型。
